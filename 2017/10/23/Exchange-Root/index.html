    树形DP之换根法

      换根法是树形DP中的一个很interesting的技巧，它解决的是在一个树形结构中，以每个节点成为一个根 进行一系列的统计的问题类。

      对于此类问题，我们有一个基本思路：

      1.进行第一次扫描，任选一个点作为定根，在有根树上进行一次树形DP，也就是在回溯的时候进行自底向上的状态转移。

      2.进行第二次扫描，对整棵树进行一次DFS，推导计算出父节点与子节点的关系，通过转移来“换根”。

    这就是解决此类问题的 二次扫描法 。

    简单看几道例题。

      有一个树形的水系，由N-l条河道和N个交叉点组成。我们可以把交叉点看作树中的节点，编号为1、N，河道则看作树中的无向边。每条河道都有一个容量，连接x与y的河道的容量记为c (x, y) 。河道中单位时间流过的水量不能超过河道的容量。

      有一个节点是整个水系的发源地，可以源源不断地流出水，我们称之为源点。除了源点之外，树中所有度数为1的节点都是入海口，可以吸收无限多的水，我们称之为汇点。也就是说，水系中的水从源点出发，沿着每条河道，最终流向各个汇点。

      在整个水系稳定时，每条河道中的水都以单位时间固定的水量流向固定的方向。除源点和汇点之外，其余各点不贮存水，也就是流入该点的河道水量之和等于从该点流出的河道水量之和。整个水系的流量就定义为源点单位时间发出的水量。
    在流量不超过河道容量的前提下，求哪个点作为源点时，整个水系的流量最大，输出这个最大值。N<=200000。

      虽说本题不一定用换根法AC，但仍然是此类问题比较好的解法。

      我们设D[x]为以x为根的子树中，把x作为源点，从x出发流向该子树的最大流量。

    当子节点y不是叶节点时，为了保证合法性，我们需要在D[y]与c（x，y）中选择一个比较小的与D[x]相加。

而当y是叶节点时，我们直接加上c（x，y）即可。

      我们再设F[x]为把x作为源点时，流向整个水系的最大流量。对于根节点root，显然有F[root] = D[root]。之后我们就能从这个root进行扩展了。

    对于一个点x，我们枚举它的子节点y。

此时，我们将整棵树划分为包含y的子树，和除y子树的这一部分。

若x为叶节点，那么F[y]等于以y为根的子树的流量D[y]加上c（x，y），即从y流向x的部分。

若x不为叶节点，那么F[y]不仅需要加上以y为根的子树的流量D[y]，还需要加上在以x为根的子树除去以y为根的子树的流量，这一部分一定要保证合法，所以表示为min（F[x]-min(D[y],c(x,y)),c(x,y)）;

    解：

    题目简化为:求一棵树上的所有路径之和。

    1.首先我们固定根，这样以该根为起点的路径和就可以全部求出来，求法就是各个路径乘以它被遍历的次数。由于终点都是叶节点，故一条边被遍历的次数取决于它所指向的子树的叶节点个数（可以画图）。这样，我们就可以在一次DFS中求出在固定根下的路径和。

    2.在第一次扫描的时候，我们要预处理出:size[x]——以x为根节点的叶子数、以该固定根为根的路径和sum。我们设T[x]为x根时整棵树的路径和，显然T[1] = sum。

    3.接下来我们开始换根。对子、父节点互相换根时，我们可以发现一条很重要的性质：以子节点为根的子树的路径和是不变的。除了这个子树以外的点所构成的子树的路径和也是不变的。这主要因为叶子的数量没有变化。唯一变化的就是子、父节点之间的边方向发生了变化，从而影响了答案。所以由此可知:
